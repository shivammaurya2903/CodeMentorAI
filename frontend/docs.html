Configuration & Security

Set these environment variables in backend/.env before starting the backend.

GROQ_API_KEY=your_groq_api_key
PORT=5000
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret
GITHUB_CALLBACK_URL=http://localhost:5000/auth/github/callback
SESSION_SECRET=your_long_random_secret

Security Best Practices

  • Never commit real API keys or OAuth secrets to git.
  • Use a strong, random SESSION_SECRET in production.
  • Rotate secrets immediately if accidentally exposed.
  • Use HTTPS and explicit frontend origins in production deployments.

Code Review Guide

Example Review

// Original code
function findDuplicates(arr) {
  const result = [];
  for (let i = 0; i < arr.length; i++) {
    for (let j = i + 1; j < arr.length; j++) {
      if (arr[i] === arr[j]) {
        result.push(arr[i]);
      }
    }
  }
  return result;
}

AI Feedback: O(n²) complexity detected. Here's an optimized O(n) solution using hashmap:

// Optimized version ✅ Score: 95/100
function findDuplicates(arr) {
  const hash = {};
  const duplicates = [];
  
  for (const num of arr) {
    if (hash[num]) {
      duplicates.push(num);
    } else {
      hash[num] = true;
    }
  }
  return [...new Set(duplicates)];
}

Troubleshooting

Backend does not start

Check that required env vars are present, especially GROQ_API_KEY. Then run backend from the correct folder:

cd backend
npm install
npm start

GitHub shows "Not Connected"

  • Verify GITHUB_CLIENT_ID and GITHUB_CLIENT_SECRET.
  • Ensure callback URL in GitHub app matches backend callback route.
  • Confirm frontend and backend are running on expected local ports.

AI responses fail

  • Validate key format and quota in your Groq account.
  • Use GET /health to verify backend status and config flags.
  • Open browser console/network tab to inspect API error details.
